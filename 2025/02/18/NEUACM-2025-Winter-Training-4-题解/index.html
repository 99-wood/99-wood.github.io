D 运输计划

题意

一个 nnn 个点的树，每条边的边权，给出 mmm 条路径。要求只改变一条边的边权，使得所有路径中长度最大的路径的长度最小。

n,m≤3×105n, m \le 3 \times 10^5n,m≤3×105。

分析

“最大值最小”提示我们要二分答案。假设我们当前要求最大路径长度不能超过 ddd。那么所有长度大于 ddd 的路径中都要包含操作的边，且操作的边的边权应当大于等于该路径的原长与 ddd 的差。
更进一步，这条边的边权应当大于等于所有路径长度最大值与 ddd 的差。所以，我们可以树上差分来给超过 ddd 的路径打标记，然后去遍历每一条边，查看是否存在一条边被所有的标记覆盖且边权大于等于路径长度最大值与 ddd 的差。

E 通配符匹配

题意

几乎所有操作系统的命令行界面（CLI）中都支持文件名的通配符匹配以方便用户。最常见的通配符有两个，一个是星号（*），可以匹配 0 个及以上的任意字符：另一个是问号（?），可以匹配恰好一个任意字符。现在需要你编写一个程序，对于给定的文件名列表和一个包含通配符的字符串，判断哪些文件可以被匹配。

对于 100%100 \%100% 的数据

  • 字符串长度不超过 100000100000100000
  • 1≤n≤1001 \le n \le 1001≤n≤100
  • 通配符个数不超过 101010

分析

注意到通配符个数不超过 101010，所以我们尝试从这里入手。首先我们可以把模式串用通配符分割成 hhh 段，其中每一段为确定的字符串或第一个字符为通配符剩下的字符为字符串。例如 a*b?c 会被分割为 a、*b 和 ?c 三部分。

定义状态 fk,if_{k, i}fk,i​ 表示待匹配串在第 iii 个字符处能否匹配到第 kkk 段结尾。

接着考虑如何转移。我们用 str\mathrm{str}str 表示模式串， sss 表示待匹配串，posk\mathrm{pos}_kposk​ 表示第 kkk 段结尾位置，lenk\mathrm{len}_klenk​ 表示第 kkk 段长度。

若第 kkk 段为确定的字符串，那么直接匹配再从前一段转移即可，即 fk,i=[fk−1,i−lenk∧strposk−lenk+1,posk=si−lenk+1,i]f_{k, i} = [f_{k - 1, i - \mathrm{len}_k} \wedge \mathrm{str}_{\mathrm{pos}_k - \mathrm{len}_k + 1, \mathrm{pos}_k} = s_{i - \mathrm{len}_k + 1, i}]fk,i​=[fk−1,i−lenk​​∧strposk​−lenk​+1,posk​​=si−lenk​+1,i​]。

若第 kkk 段为开头为 ? 的字符串，那么直接匹配确定的部分，再跳过前一个字符转移即可，即 fk,i=[fk−1,i−lenk∧strposk−lenk+2,posk=si−lenk+2,i]f_{k, i} = [f_{k - 1, i - \mathrm{len}_k} \wedge \mathrm{str}_{\mathrm{pos}_k - \mathrm{len}_k + 2, \mathrm{pos}_k} = s_{i - \mathrm{len}_k + 2, i}]fk,i​=[fk−1,i−lenk​​∧strposk​−lenk​+2,posk​​=si−lenk​+2,i​]。

若第 kkk 段为开头为 * 的字符串，那么直接匹配确定的部分，再跳过若干字符转移即可，即 fk,i=[∃(j≤i−lenk+1∧fk−1,j)∧strposk−lenk+2,posk=si−lenk+2,i]f_{k, i} = [\exists(j \le i - \mathrm{len}_k + 1 \wedge f_{k - 1, j}) \wedge \mathrm{str}_{\mathrm{pos}_k - \mathrm{len}_k + 2, \mathrm{pos}_k} = s_{i - \mathrm{len}_k + 2, i}]fk,i​=[∃(j≤i−lenk​+1∧fk−1,j​)∧strposk​−lenk​+2,posk​​=si−lenk​+2,i​]。

匹配用字符串哈希即可。

I Mex Game

题意

给定一个长度为 n+1(1≤n≤2×105)n+1(1 \le n \le 2 \times 10^5)n+1(1≤n≤2×105) 的字符串 sss。共 nnn 轮游戏，第 iii 轮进行 iii 次操作，每轮均由 Alice 先，与 Bob 交替往集合 XXX 中加入非负整数（一次加入即一次操作）。对于每一轮，记 x=mex(X)x=\mathrm{mex}(X)x=mex(X)，若 sxs_xsx​ 为 A\texttt{A}A 则 Alice 胜，反之 Bob 胜。对于每一轮，输出谁胜。

分析

集合 AAA 中的数不可能成为 xxx，所以两人的策略都是不断加入下一个另一个人的字母出现的位置。又因为操作 iii 次后，xxx 可能出现的范围为 [0,i][0, i][0,i]。所以我们只需要统计 [0,i][0, i][0,i] 中 A\texttt{A}A 和 B\texttt{B}B 出现的次数。如果对方字母出现的次数比自己能操作的次数多则输。
