CCPC2022威海站 B题 Recruitment 题解

CCPC2022威海站 B题 Recruitment 题解

题目大意

原题链接

初始有一个由 nnn 个正整数组成的形如 a1+a2+⋯+ana_1 + a_2 + \cdots + a_na1​+a2​+⋯+an​ 的式子, n−1n - 1n−1 个操作每次将其中一个加号改为乘号. 现给出初始的式子和每次修改后的式子的值 sis_isi​, 求原来的 nnn 个正整数以及每次操作的符号的位置. 输出任意方法即可, 无解输出 −1-1−1.

数据范围： n≤105,si≤109n\leq10^5, s_i \leq 10^9n≤105,si​≤109.

提示

Hint 1

sns_nsn​ 为所有数的积

Hint 2

可以根据 sns_nsn​ 和 sn−1s_{n - 1}sn−1​ 求出最后一次修改的加号两侧的数的积

Hint 3

上面的 Hint2 可以接着推到 n−1,n−2,⋯n - 1, n - 2, \cdotsn−1,n−2,⋯

Hint 4

若 si−1=si+1s_{i - 1} = s_i + 1si−1​=si​+1 说明什么?

解法

如果我们将一次修改理解为将加号两侧的数合并, 合并的数为原来两数的乘积, 就会发现问题实际是求一个可重集使得每次合并后集合中数的和等于 sis_isi​. 显然最后集合中只剩下 sns_nsn​. 则最后一次合并前的两个数设为 a,ba,ba,b 满足 a+b=sn−1,ab=sna + b = s_{n - 1}, ab = s_na+b=sn−1​,ab=sn​, 可以解方程求出这两数, 把这两个数放入集合. 考虑倒推, 我们每次枚举集合中的数尝试还原, 若成功则接着去找下一个要还原的数, 直到全部还原. 若当前集合中每个数都不能被还原, 说明我们当前的集合是错误的, 需要回退. 这一过程可以搜索解决. 但是 n≤105n \leq 10^5n≤105, 直接搜索复杂度很高. 注意到若 si−1=si+1s_{i - 1} = s_i + 1si−1​=si​+1 说明原来的两个数至少有一个是 111. 我们将 sss 中满足上述条件的剔除, 则剩下的操作前的两个数至少有一个大于等于 222, 搜索层数会降到 log⁡2sn=30\log_2{s_n} = 30log2​sn​=30 层. 实际根本没有这么高.

还可以继续剪枝, 若当前用集合中某个数 aia_iai​ 分解后失败, 则当前层不需要再尝试用别的与 aia_iai​ 相等的值去分解, 可以用 std::map 记录是否还原过某个数剪枝.
